# CLAUDE.md — Bol (Chrome extension: synced lyrics for YouTube)

Read [PROJECT.md](PROJECT.md) for architecture and how the pieces fit (start
there before any non-trivial change). Read [GAPS.md](GAPS.md) for the known
bugs, debt, and ready-scoped fix tasks, ordered by severity.

## Commands

Node is NOT on PATH on this machine. Start every shell with:

```bash
export PATH="$HOME/.local/node/bin:$PATH"   # node v24.18.0
```

Then, from the repo root:

```bash
npm run build        # full production build into dist/ (clean + js + css + static)
npm run watch:js     # esbuild watch — sets __DEV__=true (verbose logging)
npm run watch:css    # tailwind watch
npx tsc --noEmit     # typecheck — NOT part of build; esbuild does not check types. Run it yourself.
node scripts/make-icons.mjs   # regenerate public/icons/ from icons/logo-bird.png (needs sharp)
```

- There are **no tests and no linter** yet (GAPS.md #1, #8). Verification is
  manual: build, then reload the extension at `chrome://extensions` AND
  refresh the YouTube tab (both steps, always).
- `watch:js` does **not** copy `manifest.json`/icons — run a full
  `npm run build` once before the first load-unpacked, then watch.
- Pipeline diagnostics (every query, score, rejection reason) log to the
  **background service worker console**: `chrome://extensions` → Bol →
  "service worker". Panel/UI logs are in the tab console, only in watch builds.
- Deploy = zip the contents of `dist/` (top-level, no wrapper dir, exclude
  `*.map`) and upload to the Chrome Web Store. Store copy lives in
  `STORE_LISTING.md`.

## What is generated / source of truth

- `dist/` — generated, never edit. `public/manifest.json` is the real manifest.
- `public/icons/*.png` — generated by `scripts/make-icons.mjs`; edit the
  script or `icons/logo-bird.png`, not the PNGs.
- Extension version = `public/manifest.json` `"version"` (package.json version
  is stale legacy — see GAPS.md #9).
- `PRD.md`, `ARCHITECTURE.md`, `DATA_FLOW.md`, `EXTENSION_SPEC.md` are
  **historical pre-implementation design docs** — they describe features that
  were never built (AI drift correction, karaoke mode, extra permissions).
  Do not implement against them. README.md, CHANGELOG.md, PROJECT.md are accurate.

## Conventions this codebase actually follows

- TypeScript strict everywhere; React function components + hooks only.
- Comments explain **why** (regressions, measurements, UX rules), not what.
  Keep that density when editing — thresholds and regexes especially carry
  their justification inline. Don't strip these comments.
- Log prefix is `[adaptive-lyrics]` (+ `[lrclib]`/`[netease]`/`[panel]` scopes).
  Content-script lifecycle logs use `dlog()` from `src/lib/debug.ts`
  (compiled out in production via `__DEV__`). Provider/background logs use
  bare `console.info` **intentionally** — they're the only production
  diagnostics. Warnings/errors are never gated.
- All YouTube DOM selectors live in `src/lib/domSelectors.ts`, always scoped
  under the watch-page root — never `document.querySelector` YouTube markup
  anywhere else, never unscoped (YouTube reuses `#title` in popups).
- Lyrics providers implement `LyricsProvider` (`src/lib/providers/types.ts`):
  new provider = new file in `src/lib/providers/` + append to `PROVIDERS` in
  `providers/index.ts` + add the API host to `host_permissions` in
  `public/manifest.json`. Return `{ lyrics: null }` for "no match"; throw only
  for unexpected failures.
- Network calls go through `timedFetchJson` (`src/lib/providers/http.ts`) —
  typed outcomes (`ok`/`timeout`/`http-error`/`network-error`), never bare fetch.
- All rendered strings are React text nodes. Never introduce
  `dangerouslySetInnerHTML` — lyric/title data is third-party.
- Styling: Tailwind classes with `al-` prefixed custom classes in
  `src/content/styles.css`; everything renders inside a shadow root.

## Gotchas (things that look wrong but aren't, and vice versa)

- **The panel never remounts.** Mounted once per tab; open/close is a state
  machine in `App.tsx` (`closed→opening→open→closing`) driving pure
  transform/visibility. Don't "fix" by conditional-rendering the panel or
  remounting on navigation — that was a real bug (flicker, lost state).
- **Video changes must never open/close the panel.** Opening is
  user-initiated only (toolbar, thin tab, Alt+L). Don't add auto-open.
- **Open/closed state is deliberately not persisted.** Persisting it via
  `chrome.storage.sync` caused cross-tab toggling races. Only `displayMode`
  and `panelWidth` are stored.
- **`REQUEST_TIMEOUT_MS = 12000` is measured, not arbitrary.** LRCLIB under
  our 8-parallel-query load takes 5–10s; Chrome queues past 6
  connections/host and queue time counts toward the timeout. It was 5000 once
  and silently killed most lookups. Do not lower without re-measuring.
- **NetEase needs `lv=-1`** on the lyric endpoint; the "documented" `lv=1`
  returns empty for many songs. If NetEase stops matching, manually test the
  endpoint before touching code — the API is unofficial and can die silently.
- **Only positive lyrics results are cached** (`chrome.storage.session`).
  Caching nulls locked findable songs into "unavailable" for a session.
  Keep it that way.
- **Providers fire all queries in parallel but evaluate in priority order.**
  The two-phase structure in `lrclib.ts`/`netease.ts` looks redundant; it's a
  deliberate latency optimization with sequential selection semantics. Keep
  selection order stable if you touch it.
- **Scoring thresholds encode regressions**: `FLOOR_SCORE` 0.5 (was 0.3 — the
  "Bairan" wrong-match), `ACCEPT_SCORE` 0.55, canonical bar 0.8. Wrong lyrics
  are worse than no lyrics. Any threshold change needs manual testing against
  the songs named in the code comments.
- **Extension reload orphans old content scripts.** `chrome.runtime?.id`
  guards and try/catch around chrome APIs throughout are load-bearing;
  `content.tsx` replaces a leftover host element on purpose. Keep the guards.
- **`e.key` vs `e.code`**: the Alt+L handler currently checks `e.key === 'l'`,
  which likely never fires on macOS (Option+L composes `¬`) — GAPS.md #3.
- **Stale-response race** in the `App.tsx` fetch effect can show the previous
  video's lyrics — GAPS.md #2. Check these two before debugging "wrong panel
  behavior" from scratch.

## Rules (do not violate)

1. **Never rename** the storage keys (`adaptive-lyrics-settings`,
   `lyrics:<videoId>` in session storage) or the host element id
   (`adaptive-lyrics-host`) — renames lose user settings / break orphan
   cleanup. The `adaptive-lyrics` internal codename vs "Bol" product name
   split is known and accepted.
2. **Never show wrong lyrics.** Any matching change that could accept a
   looser match must add corroboration (fragment + duration — see
   `src/lib/providers/guards.ts`), not just lower a threshold.
3. New network hosts require both a `host_permissions` entry in
   `public/manifest.json` **and** an update to `PRIVACY.md`/`STORE_LISTING.md`
   (store-review consistency).
4. Keep `manifest.json` permissions minimal — currently just `storage`; no
   `tabs`, no `activeTab`, no `scripting` are needed. Don't add "just in case".
5. Run `npx tsc --noEmit` before considering any change done — the build
   passing does not mean the types are sound.
